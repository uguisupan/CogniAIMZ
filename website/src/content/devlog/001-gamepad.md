## 1. はじめに：なぜ「単一目的」なのか

モータースポーツにおける初期トレーニングは、多くの場合、実車や総合シミュレーターでの走行から始まります。しかし、これは「最初からマルチタスク（ステアリング、シフト、ブレーキ、挙動変化、周辺認知など）を同時に処理できる人間」以外をふるい落とす「適性検査」になってしまっています。

本サークル「CogniAIMZ」では、最も脳の認知資源を食い潰しやすく、かつ感覚のズレが起きやすい「ブレーキペダルのリリースコントロール」のみを切り出し、徹底的に自動化（オートマティズム）するためのツール『Brake Trainer (K2MF-001)』の開発を本日スタートしました。

まずは基本となるプロトタイプ **v0.1.0** を実装しました。

---

## 2. v0.1.0 の実装内容

* **Gamepad API によるペダル入力の取得**
  ブラウザ標準 of Gamepad API を利用して、PCに接続されたロードセル式ペダル（Fanatec等）のアナログ入力をミリ秒単位で直接ポーリングする処理を実装。
* **リズムゲーム型判定エンジン**
  画面上のターゲットラインに対し、指定されたテンポで踏力（％）を追従させる基本的なリズム判定ロジックを実装。

## 3. 小国マイ・小国ハナミ両名による解説協力

このプロジェクトを進めるにあたり、当サークルが研究する難解な運動学習理論やシステムを、標準的ドライバーや大きなお友達にわかりやすく解説してくださる「インタープリター」として、メイドの小国マイ・小国ハナミ両名にご協力いただいてます。

役割分担として、小国ハナミ氏が「クソ難しい理論やペダル制御の物理を無理やり叩き込む技術指導（お姉さん役）」を、小国マイ氏が「それらを噛み砕いて親しみやすく伝えるインタープリター（案内役）」を担当します。

開発初期段階ではガチガチの数式と生データだけでしたが、マイ氏の「難しすぎて誰も触らなくなります！」という進言と、ハナミ氏の「まずは身体でタイミングを覚えさせる音ゲー式ギブスから始めなさい」というアドバイスを取り入れ、現在のUI設計の方向性が定まりました。
